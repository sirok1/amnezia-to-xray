# amnezia-to-xray
## Поток трафика
 
```
Клиент AWG (10.8.1.x)
         │
         │ WireGuard / AmneziaWG
         ▼
┌─────────────────────────────────────────────────┐
│           amnezia-awg2 (контейнер)              │
│                                                 │
│  awg0 (10.8.1.0/24)                             │
│    │                                            │
│    │  ip rule: from 10.8.1.0/24 → table 200     │
│    ▼                                            │
│  tun1 (198.18.0.2)                              │
│    │  table 200: default → tun1                 │
│    │                                            │
└────┼────────────────────────────────────────────┘
     │
     │  tun2socks (хост, nsenter в netns контейнера)
     │  socks5://172.29.172.1:12346
     │
     ▼
┌─────────────────────────────────────────────────┐
│           3x-ui / X-ray (host network)          │
│                                                 │
│  inbound: amnezia-in                            │
│  port: 12346, protocol: socks5                  │
│                                                 │
│              Роутинг X-ray                      │
│                    │                            │
│       ┌────────────┼────────────┐               │
│       │            │            │               │
│       ▼            ▼            ▼               │
│      rule         rule        direct            │
│       │            │            │               │
│       ▼            ▼            ▼               │
│   outbound      outbound      outbound          │
│   (socks5)       WARP WG     (этот сервер)      │
└─────────────────────────────────────────────────┘
```
 
## Исключения
 
| Трафик | Обработка |
|--------|-----------|
| DNS (порт 53) | Напрямую → `amnezia-dns` (172.29.172.254) |
 
## Компоненты
 
| Компонент | Роль |
|-----------|------|
| `amnezia-awg2` | Принимает WG клиентов, ip rule → tun1 |
| `tun2socks` | Заворачивает трафик tun1 в socks5 |
| `3x-ui / X-ray` | Принимает socks5, применяет роутинг |
| `amnezia-dns` | DNS для клиентов (172.29.172.254) |

## пример инбаунда сокс

```JSON
....
{
      "tag": "amnezia-in",
      "listen": "172.29.172.1",
      "port": 12346,
      "protocol": "socks",
      "settings": {
        "udp": true,
        "auth": "noauth"
      },
      "sniffing": {
        "enabled": true,
        "destOverride": [
          "http",
          "tls",
          "quic"
        ]
      }
    }

```

Ну и естественно надо настроить правила маршрутизации, балансера (если планируете использовать больше 1 прокси ) и аутбаунды
